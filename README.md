The **Hospital Query Routing System** is a backend-driven web application designed to efficiently manage and route patient queries across various hospital departments using predefined workflows. The system allows patients or administrators to create service requests such as lab tests, scans, appointments, pregnancy consultations, and general checkups. Once a request is submitted, it is automatically routed to the appropriate department, and its movement is tracked at every stage. The application ensures transparency by maintaining detailed status updates such as Pending, In Progress, Approved, Rejected, or Completed, along with a complete history of department transfers and timestamps.

The system is developed using **HTML5, CSS3, and JavaScript**, where JavaScript handles the workflow logic, status transitions, and request tracking mechanisms. It provides structured patient records with complete details including patient information, request category, assigned department, and progress updates. The system also supports downloading patient records in a clear, line-by-line format for better readability and documentation. This project is suitable for academic demonstrations and can be further extended with database integration, authentication modules, or real-time notifications to make it production-ready.

To run this project locally, it is recommended to use **Visual Studio Code** with the **Live Server extension** installed. After installing Visual Studio Code, add the Live Server extension from the Extensions marketplace. Once installed, open the project folder in Visual Studio Code and right-click on the `index.html` file, then select **“Open with Live Server.”** This will launch the application in your default web browser with automatic reload functionality whenever changes are made to the code. Using Live Server ensures smooth development, real-time preview, and easier debugging during project execution.
